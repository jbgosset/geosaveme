# Design

Project framing documents — the "what and why" before the "how".

## Contents

| Folder | Purpose |
|--------|---------|
| `customers-users/` | Customer segments, user personas, user journeys |
| `functional-scope/` | High-level feature scope, MVP definition, roadmap |
| `use-cases/` | Use case narratives and diagrams |
| `architecture/` | System architecture overview (non-technical, decision rationale) |
| `ux-guidelines/` | Design system, UI principles, accessibility, mobile UX standards |
| `mobile-policies/` | App Store / Play Store policies, platform constraints |
| `legal/` | Privacy policy drafts, terms of service, GDPR / CCPA compliance |
| `schedule/` | Project timeline, milestones, sprint plan |
| `testing-strategy/` | Overall testing philosophy, QA approach, coverage targets |
| `infrastructure/` | Infrastructure principles (cloud-agnostic, multi-region, cost model) |
| `methodology/` | Development methodology, branching strategy, AI-human collaboration rules |
