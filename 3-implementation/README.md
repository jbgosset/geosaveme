# Implementation

Detailed specifications — the "how". This is what AI agents and developers read before coding.

## Contents

| Folder | Purpose |
|--------|---------|
| `functional-specs/` | Detailed functional specifications per feature/service |
| `technical-guidelines/` | Technical spec, API contracts, data models |
| `architecture/` | Detailed architecture diagrams, service dependency maps |
| `infrastructure/` | Cloud config, Kubernetes, database specs, CDN, monitoring setup |
| `cicd/` | CI/CD pipeline rules, deployment gates, environment promotion |
| `testing-scenarios/` | Test scenarios per feature — required before any code is "done" |
| `agent-instructions/` | **Start here** — rules for all AI agents and contributors |

## Rule

No code is written without a corresponding document in this folder.  
No document is written without being linked to a spec or design decision.
